Documentation for MyFlix API v1.0.0

The API is the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

The API is built with REST architecture with multiple routes for the REST API, myFlix, so that clients can apply operations (creating, reading, updating, and deleting, i.e., CRUD) to files and data on the server.

Errors are returned as standard HTTP response codes. Messages about error are included in the response body.

Request URL HTTP Method The query parameters Request body data format Response body data format
Get a list of all movies /movies GET Required (JWT) None JSON array of movie objects
Get data about a movie by name /movies/title/:name GET Required (JWT) None JSON response holding data of the movie required by name. For example: { "title": "21 JUMP STREET", "starring": "Jonah Hill, Channing Tatum, Brie Larson, Dave Franco", "director": "Phil Lord, Chris Miller", "year": 2012, "genre": "comedy" },
Get data about movies by genre /movies/genre/:genre GET Required (JWT) None JSON response holding data of the movie required by genre.
